# IMechE Formula Student Rules - Learning & Revision Guide

Welcome to the **IMechE Formula Student (FS) Rules & Revision Guide** repository. This project is a group-driven, centralized resource designed to translate the dense, legalistic official Institution of Mechanical Engineers (IMechE) Formula Student rulebook into a structured, highly accessible, and digestible "revision guide" format.

Our goal is to ensure that team members—from first-year recruits to seasoned engineers—can quickly master compliance requirements, understand technical constraints, and excel in competitive strategy.

---

## 📌 Purpose & Objectives

The official Formula Student rules are comprehensive but can be overwhelming to navigate under tight design and manufacturing deadlines. This repository acts as a bridge between the raw rules and practical engineering application.

- **Accelerate Learning:** Help new team members onboard and understand standard vehicle constraints rapidly.
- **Rule Memorization & Mastery:** Provide active recall tools and summaries for the mandatory rules quiz.
- **Scrutineering Success:** Highlight critical "fail-points" to ensure the vehicle passes technical inspection on the first attempt.
- **Competitive Edge:** Break down the scoring system of both static and dynamic events to optimize design choices for maximum points.

---

## 🗂️ Repository Structure

The documentation is broken down into modular, logical sections matching standard Formula Student yearly formats.

Generic scheme of organisation is as follows:

```text
├── .github/                 # Issue templates and workflows
├── '~year~/                 # Docment release year Directory
|    ├── FSUK-~year~-Study-Guide.html         # Revision guide like document
|    ├── fsuk-~year~-rules-sup.html           # Official supplementry rules document
|    └── fsuk-~year~-rules.html               # Official supplementry rules document
└── extra_documents/          # Additional relevant documententation or anything interesting 
```
